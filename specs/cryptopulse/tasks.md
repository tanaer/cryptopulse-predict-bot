# 实施计划

- [ ] 1. 初始化单仓多应用工程骨架
  - 创建 `apps/admin`（Next.js App Router + shadcn/ui）
  - 创建 `apps/bot`（grammY）
  - 创建 `packages/db`（Prisma）与 `packages/polymarket`（SDK 封装）
  - 增加 Docker Compose（Postgres + Redis）
  - _需求: 非功能需求_

- [ ] 2. 实现单管理员后台与基础页面框架
  - 单管理员登录/鉴权（ADMIN_TOKEN）
  - 后台导航与基础页面：统计、推送、用户、黑名单、配置
  - _需求: 需求 7、需求 10、需求 11_

- [ ] 3. 实现用户绑定闭环（嵌入式优先 + 兜底签名）
  - 优先集成 Privy + Safe 或 Magic Link + Safe 的一键绑定（复用官方 Telegram 示例）
  - 兜底：Bot 生成 bind_code → Web 绑定页钱包连接 + 签名 → 服务端验证写入用户绑定信息
  - 增加 Week 1.5 检查点：在测试群完成无缝绑定验证，并对比 Privy vs 兜底签名 UX（可观测流失）
  - _需求: 需求 1_

- [ ] 4. 实现市场发现与详情（含缓存与实时刷新）
  - 市场搜索、分类列表与分页
  - 市场详情：订单簿/价格/成交量等
  - 优先实现 CLOB WebSocket 订阅模块（降级轮询）
  - Redis 缓存与限流退避
  - _需求: 需求 2、需求 3、需求 11_

- [ ] 5. 实现交易下单（市价/限价）与归因签名服务
  - 封装 CLOB trading client（BuilderConfig remote signing）
  - 下单流程：输入金额→确认页→执行→回执
  - 首次交易自动检查 safe_address，缺失则通过 Relayer 部署/初始化并给出进度提示
  - 签名端点 `POST /api/polymarket/sign`（SIGNING_TOKEN）
  - 归因验证任务（队列/定时）与后台展示
  - Unverified 阶段提示与里程碑：完成 10-20 笔端到端测试交易后提交 Verified 申请邮件
  - _需求: 需求 4、需求 7、需求 11_

- [ ] 6. 实现仓位/订单与 CTF 操作（Split/Merge/Redeem）
  - 仓位与订单查询、缓存与展示
  - Relayer client 执行 CTF 操作并回执
  - _需求: 需求 5、需求 11_

- [ ] 7. 实现通知与自动化（热点推送/警报/早报）
  - 后台触发热点推送（PushJob）
  - 用户价格警报与触发器
  - 定时早报生成与推送
  - _需求: 需求 6、需求 11_

- [ ] 8. 实现复制交易（可用版 + 风控）
  - 跟单配置、启停、风控规则（最大单笔/最大仓位/黑名单）
  - 事件来源优先从管理员后台手动发布“复制信号”（可二次确认或自动下单）
  - _需求: 需求 8、需求 11_

- [ ] 9. 实现 AI 预测助手（可插拔 Provider）
  - 市场详情 “AI 解读” 输出（先默认模板化摘要，可接入 LLM）
  - 频道内容观点草稿（管理员审核发布）
  - _需求: 需求 9、需求 11_

- [ ] 10. 完成 Web Dashboard 与中英双语
  - Dashboard 图表与数据接口
  - Bot 消息模板 + 后台 UI 文案 i18n
  - _需求: 需求 10_

- [ ] 11. 验收与交付
  - 关键路径自测：绑定→搜索→详情→下单→仓位→CTF→推送→归因验证
  - 增加基础测试与运行手册（环境变量清单、测试群 Bot E2E、归因验证自动化脚本）
  - _需求: 全部_
